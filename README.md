# Project CS

> A long-term, hands-on journey from beginner programmer to technical founder.

Project CS is a personal software engineering project that will grow alongside my programming knowledge. Its purpose is not only to teach me how to code, but also to document how I learn to design, build, test, debug, and improve real software.

The project will begin as a simple website and gradually evolve into a gamified programming-learning platform built through the same skills it teaches.

## Current Status

**Project stage:** Early development  
**Current sprint:** Sprint 2 — HTML Foundation  
**Current focus:** Building a clean, semantic website structure before adding styling or interactivity

The repository is intentionally simple right now. The goal is to understand each technology deeply instead of rushing into frameworks or copying finished solutions.

## The Vision

Project CS will eventually feel more like a video game than a traditional educational website.

Instead of only reading tutorials, the learner will:

- Complete programming quests
- Build real projects
- Fight topic-based boss battles
- Unlock meaningful achievements
- Track levels and progress
- Use an embedded code editor
- Receive AI-generated hints and feedback
- Maintain a developer journal

Each major subject will become its own world, such as:

- Python Academy
- HTML Kingdom
- CSS Forge
- JavaScript Laboratory
- SQL Vault
- C Fortress
- C++ Mountain
- AI Nexus
- Game Development Arena

## Core Learning Philosophy

Project CS follows a build-first learning system:

1. Learn a concept.
2. Explain why it works.
3. Practice it with small exercises.
4. Use it in a real feature or project.
5. Test and debug the result.
6. Commit the work with Git.
7. Reflect on what was learned.
8. Continue using the skill in future projects.

New technologies will not replace older ones. Each new skill should build on and reinforce everything learned before it.

## Teaching Principles

Project CS is designed to improve both programming ability and independent thinking.

The learning process should:

- Favor hints over immediate answers
- Require reasoning before implementation
- Encourage experimentation
- Treat mistakes as debugging opportunities
- Use boss battles to test independent mastery
- Prioritize understanding over speed

## Current Sprint: HTML Foundation

The current goal is to create the semantic structure for the first Project CS interface using only HTML.

Planned sections include:

- Site header and navigation
- Main game-style menu
- Continue Journey panel
- Current quest
- Progress and level information
- Programming worlds
- Recent achievements
- Developer journal preview
- Project footer

CSS and JavaScript will be added only after the HTML structure is understandable, semantic, and accessible.

## Development Workflow

Every meaningful feature should follow this cycle:

```text
Think → Sketch → Plan → Build → Test → Debug → Commit → Reflect
```

Git commits should represent clear, understandable progress rather than being created only to increase the commit count.
